Scenes¶

Scenes are what can be considered the canvases for your animations. All scenes are derived from the Scene class, which in itself is derived from the Container class.

Here’s a quick rundown of how manim processes videos:

  1. Get classes deriving from Scene.

  2. Execute the digest_config function to associate parameters for the scene.

  3. Execute the scene’s construct function and render whatever’s in it.

Let’s take a look at the pre-defined scenes in manim. (Yes you can create your own as well!)

Scene¶

class manimlib.scene.scene.Scene(**kwargs)¶

The base class for all scenes.

This class contains lots of useful functions, though most of the time you’ll be fine with only Scene.play() and Scene.wait()

add(*mobjects)¶

Mobjects will be displayed, from background to foreground in the order with which they are added.

add_mobjects_among(values)¶

This is meant mostly for quick prototyping, e.g. to add all mobjects defined up to a point, call self.add_mobjects_among(locals().values())

add_sound(sound_file, time_offset=0, gain=None, **kwargs)¶

Play the sound_file.

compile_play_args_to_animation_list(*args, **kwargs)¶

Each arg can either be an animation, or a mobject method followed by that methods arguments (and potentially follow by a dict of kwargs for that method). This animation list is built by going through the args list, and each animation is simply added, but when a mobject method s hit, a MoveToTarget animation is built using the args that follow up until either another animation is hit, another method is hit, or the args list runs out.

construct()¶

The function looked for by manim while rendering a Scene.

get_animation_time_progression(animations)¶

The method defining the ProgressBar you see on a render.

get_moving_mobjects(*animations)¶

Go through mobjects from start to end, and as soon as there’s one that needs updating of some kind per frame, return the list from that point forward.

get_time()¶
get_time_progression(run_time, n_iterations=None, override_skip_animations=False)¶

Used by the renderer.

get_top_level_mobjects()¶

Return only those which are not in the family of another mobject from the scene

handle_play_like_call()¶

A wrapper for implementing Scene.play() and Scene.wait()

print_end_message()¶

The message one sees at the end of a render.

progress_through_animations(animations)¶

Paint all non-moving objects onto the screen, so they don’t have to be rendered every frame

remove(*mobjects)¶

To remove Mobjects in the reverse order they were ``Scene.add``ed.

restructure_mobjects(to_remove, mobject_list_name='mobjects', extract_families=True)¶

In cases where the scene contains a group, e.g. Group(m1, m2, m3), but one of its submobjects is removed, e.g. scene.remove(m1), the list of mobjects will be editing to contain other submobjects, but not m1, e.g. it will now insert m2 and m3 to where the group once was.

set_variables_as_attrs(*objects, **newly_named_objects)¶

This method is slightly hacky, making it a little easier for certain methods (typically subroutines of construct) to share local variables.

setup()¶

This is meant to be implement by any scenes which are comonly subclassed, and have some common setup involved before the construct method is called.

Graph Scene¶

class manimlib.scene.graph_scene.GraphScene(**kwargs)¶

Creates a customizable Cartesian (x-y) axis system. Used mainly to draw algebraic plots.

If you’re looking for the blue gridded plane shown in 3Blue1Brown’s videos, see:

  1. NumberPlane (manimlib.mobject.coordinate_systems.NumberPlane)

  2. LinearTransformationScene (manimlib.scene.vector_space_scene.LinearTransformationScene)
